34 class DataRecorder:
35  """
36  DataRecorder objects concentrate the tools for collecting profiles of
37  execution and resource usage. They support performance analysis by taking
38  care that all relevant information will be recorded.
39 
40  When a DataRecorder is created, it starts the profiler and records the
41  current time, memory consumption, and cpu utilization. When the recorder is
42  stopped, it again records the time, memory and cpu usage. After stopping
43  the recorder, the execution profile and resource usage data may be dumped
44  into a set of files for later analysis.
45 
46  For example, suppose you want to collect profiling information of an
47  algorithm implementation. The following code creates a profile and measures
48  the used resources; finally it writes them to the files @c algo.profile and
49  @c algo.timing
50 
51  @code
52  dr = DataRecorder( "algo" ) # Start recorder
53  algo( some_parameter_set ) # Invoke algorithm
54  dr.stop()
55  dr.dump() # Write the recorded information to disk
56  @endcode
57 
58  @note It is the user's responsibility to dump() the collected
59  information; a DataRecorder will not write information to the
60  disk on its own. This allows throwing away incomplete
61  data sets.
62 
63  @see The @c cProfile and @c resource modules
64  """

132  """
133  Write the collected information to the files supplied to the
134  constructor.
135 
136  Calling this method implies a call of stop(); thus data recording ends.
137 
138  @param extra_information A dictionary with additional entries to
139  the @c .timing file. Use this, for

162  datetime.now().strftime("%Y/%m/%d %H:%M:%S")

166  "sys time (s): {0}".format( sys_time ),
167  "cpu time (s): {0}".format( cpu_time),
168  "max memory (kB): {0}".format( max_rss ),

206 class ParallelParser:
207  """
208  A simple parser for "separated value" (SV) files that makes sure that every
209  line of the file is yielded in only one instance operating on it.
210 
211  Use this class, for instance, to have multiple instances of a program
212  operate on the same parameter list. The class makes sure that every line
213  is read by only one instance of the program.
214 
215  The different instances of ParallelParser synchronize their operation
216  through a shared text file. File system locking prevents concurrent reads
217  and writes. Since this works even on networked file systems, parser
218  instances can in this way communicate across a network. The communication
219  file is named @c file.pipe, where @c file is the name of the parsed file.
220 
221  To iterate over a file @c foo.txt line by line, you could use the following
222  code:
223  @code
224  for line in ParallelParser( "foo.txt" ):
225  # Do something with line
226  @endcode
227 
228  Now suppose you have a computation intensive algorithm @c f(x,y) and you
229  want to apply it once to the parameter set of every line in the parameter
230  list in @c param.lst. Thus you write a program like
231  @code
232  for params in ParallelParser( "param.lst", "," ):
233  f(*params) # Expand the tuple params
234  @endcode
235  If you start, say, two instances of the program and the contents of
236  @c param.lst are
237  @code
238  1,2
239  3,4
240  5,6
241  @endcode
242  then one instance will receive the tuple @c (1,2) and the other @c (3,4).
243  The final tuple @c (5,6) will be read by whichever instance finishes first.
244 
245  @note This class is designed for use cases where each yielded line
246  takes a long time to process. Synchronization by file locking
247  probably involves a large overhead, so fast processing with many
248  parsers at the same time will scale badly.
249 
250  @see __iter__() for a detailed description which lines the parser yields.
251  """
252 
253  def __init__(self, file, separator=" "):
254  """
255  Construct a new ParallelParser for iterating over @p file.
256 
257  @param file The name of the file to be parsed. It als serves
258  as the first part of the name of the shared
259  communication file @c file.pipe.
260  @param separator The parser splits the entries in each line of
261  @p file by @p separator. For example, to
262  retrieve a the entries of a comma separated
263  value (CSV) file as a tuple, use "," as
264  separator. The line "1,2,3" then yields the
265  tuple @c (1,2,3); with a space as separator, the
266  parser yields a tuple with one entry: the
267  string "1,2,3".
268  """
269  self.__separator = separator
270  self.__file = open( file, "rt" )
271 
272  directory = os.path.dirname( file )
273  pipe_file = "{}.pipe".format( os.path.basename( file ) )
274  self.__pipe = open( os.path.join( directory, pipe_file ), "at+" )
275  self.__locked = False

292  def __iter__(self):
293  """
294  Interface method to support line by line iteration with the @c for
295  statement.
296 
297  Every line is split with the @c separator provided to __init__(),
298  the method thus returns a tuple of contents. For example, the line
299  "1,2,3" will yield @c (1,2,3) if the @c separator is ",".
300 
301  The parser skips empty lines and comment lines, that is, lines that
302  start with zero or more spaces followed by a hash '#'.
303 
304  @return A tuple of the line contents, split by @c separator.
305  """
306  # Register: increase the number of parsers
307  with self.__lock() as data:
308  parsers, current_offset, current_line = data
309  self.__update( parsers + 1, current_offset, current_line )
310 
311  # Iterate until the file ends
312  # readline() is just at test for EOF; file will be seek()ed anyway
313  line = self.__file.readline()
314  while line:
315  with self.__lock() as data:
316  parsers, current_offset, current_line = data
317  self.__file.seek( current_offset )
318  line = self.__file.readline()
319  current_line += 1
320 
321  # Skip over empty lines and comments until the file ends
322  while line and (not line.strip() or line.strip().startswith( "#" )):
323  line = self.__file.readline()
324  current_line += 1
325 
326  # File might have ended already
327  if line:
328  self.__update( parsers, self.__file.tell(), current_line )
329  yield current_line, tuple( line.strip().split( self.__separator ) )
330 
331  # Unregister: decrease the number of parsers
332  with self.__lock() as data:
333  parsers, current_offset, current_line = data
334  self.__update( parsers - 1, current_offset, current_line )
335 
336  raise StopIteration

339  @contextmanager
340  def __lock(self):
341  """
342  Obtain a lock on the shared communication file.
343 
344  Use this @c contextmanager with the @c with statement.
345 
346  @see The @c contextlib module.
347  """
348  previously_locked = self.__locked
349  fcntl.lockf( self.__pipe, fcntl.LOCK_EX )
350  self.__locked = True
351 
352  try:
353  self.__pipe.seek( 0 )
354  parsers, offset, line = map(int, self.__pipe.readline().split( " " ) )
355  except ValueError:
356  parsers, offset, line = 0, 0, 0
357 
358  yield parsers, offset, line
359 
360  self.__locked = previously_locked
361  if not previously_locked:
362  fcntl.lockf( self.__pipe, fcntl.LOCK_UN )

365  def __update(self, parsers, offset, line):
366  """
367  Update the contents of the shared communication file, for example with
368  the new input file position after a line has been read.
369  """
370  with self.__lock():
371  self.__pipe.seek( 0 )
372  self.__pipe.truncate()
373  print( parsers, offset, line, file=self.__pipe )
374  # Flush the update to disk to circumvent caching, which might delay
375  # propagation of the update to other parsers.
376  self.__pipe.flush()

386 class AlgorithmRunner:
387  """
388  An AlgorithmRunner allows fast creation of a feature-rich command line
389  program that applies an algorithm to sets of parameters while recording
390  the execution profile.
391 
392  For example, say you have an algorithm (that is, a callable) @c f(x,y) and
393  want to apply it to every parameter set listed in the file @c params.lst.
394  Furthermore suppose you want to record the calling profile (see the
395  @c cProfile module), the execution time, and the result of each
396  application. Then a script containing the following code is your program
397  (@c prog.py):
398  @code
399  def f(x, y, output):
400  # Do calculations here
401  print( "The ultimate truth is:", result, file=output )
402  return result
403 
404  r = AlgorithmRunner( f )
405  r.run()
406  @endcode
407  You then invoke
408  @code
409  python prog.py -d profiles/ -p -i params.lst
410  @endcode
411  and it will apply @c f() to each line of @c params.lst and store the
412  respective profile and timing information in the @c profiles/ directory.
413  Use the "-h" command line switch to get a list of all available options.
414 
415  AlgorithmRunner uses ParallelParser to read the input files. It therefore
416  supports multiple program instances working on the same parameter list.
417  Each line will be read by only one instance.
418 
419  Send the signal @c SIGUSR1 to an AlgorithmRunner instance to "soft" kill
420  it. It will then properly shut down, unregistering its ParallelParser etc.,
421  all of which does not happen if it is just killed.
422 
423  @see ParallelParser and DataRecorder
424  """
