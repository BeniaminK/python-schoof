Detailed Description

An AlgorithmRunner allows fast creation of a feature-rich command line program that applies an algorithm to sets of parameters while recording the execution profile.

For example, say you have an algorithm (that is, a callable) f(x,y) and want to apply it to every parameter set listed in the file params.lst. Furthermore suppose you want to record the calling profile (see the cProfile module), the execution time, and the result of each application. Then a script containing the following code is your program (prog.py):

1 def f(x, y, output):
2  # Do calculations here
3  print( "The ultimate truth is:", result, file=output )
4  return result
5 
6 r = AlgorithmRunner( f )
7 r.run()

You then invoke

1 python prog.py -d profiles/ -p -i params.lst

and it will apply f() to each line of params.lst and store the respective profile and timing information in the profiles/ directory. Use the "-h" command line switch to get a list of all available options.

AlgorithmRunner uses ParallelParser to read the input files. It therefore supports multiple program instances working on the same parameter list. Each line will be read by only one instance.

Send the signal SIGUSR1 to an AlgorithmRunner instance to "soft" kill it. It will then properly shut down, unregistering its ParallelParser etc., all of which does not happen if it is just killed.

Parameters
algorithmA callable that takes arguments, where is the number of entries per line in parameter lists (or given on the command line). The last argument is output, which gives the algorithm a way to write logging information. The return value of algorithm will be stored alongside the timing information.
argumentsA list of command line argument strings for the runner.
algorithm_versionThe algorithm version will be stored together with the timing information. Use it to recall which revision of an algorithm was used in a particular profile.
