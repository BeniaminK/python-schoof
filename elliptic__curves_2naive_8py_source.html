32  """
33  An elliptic curve; the points support additive infix notation for group
34  operations, as well as multiplication by integers.
35 
36  This is a template class that must be instantiated with the field and the
37  parameters @c A and @c B. Use it, for example, as follows:
38  @code
39  # Instantiate the elliptic curve template as y^2 = x^3 + 3x + 4 over GF(7)
40  E = EllipticCurve( FiniteField(7), 3, 4 )
41 
42  # Create two points and add them
43  P = E(1, 1)
44  Q = E(2, 5)
45 
46  P + Q == E(6, 0) # Point addition
47  3*P == E(0, 5) # Multiplication by integers
48 
49  O = PointAtInfinity()
50  P == P + O # The point at infinity is the neutral element
51  type(P) is E # This is also true
52  @endcode
53 
54  An elliptic curve is the set of all points @f$ E = \bigl\{ (x, y) \mid y^2
55  = x^3 + Ax + B \bigr\} \cup \{\mathcal{O}\} @f$, where @f$ \mathcal{O} @f$
56  is a special element, the point at infinity. The points form an abelian
57  group with the point at infinity as neutral element.
58 
59  The relation @f$ y^2 = x^3 + Ax + B @f$ between the (Cartesian) coordinates
60  of the points @f$ (x, y) \in E @f$ is called the fundamental relation.

169  def __add__(self, other):
170  """
171  Return the sum of @p self and @p other. The infix operator @c + calls
172  this method.
173 
174  The geometric interpretation of point addition is to draw a line between
175  the points and mirror the point at the third intersection of line and
176  curve on the y-axis. However, the justification why this works is
177  rather complicated. Please see some introductory text for the formulas
178  and the respective proofs.
179 
180  @note Addition on elliptic curves is commutative. The neutral element
181  is the point at infinity (compare PointAtInfinity).
182 
183  @see For example Washington, Lawrence C. "Elliptic Curves:
184  Number Theory and Cryptography", second edition, CRC Press 2008,
185  chapter 2.
186  """
187  if other.is_infinite():
188  return self
189  elif self == -other:
190  return PointAtInfinity()
191  else:
192  if self.__x == other.x():
193  return self.__double__()
194  else:
195  return self.__generic_add__( other )

198  def __generic_add__(self, other):
199  """
200  Generic addition of points @p self and @p other: the points are neither
201  identical nor inverses.
202 
203  @note This method should not be called directly. Instead use the
204  infix operator @c +, which calls __add__().
205 
206  @note Putting the cases in separate methods makes call profiles more
207  expressive.
208 
209  @see __add__()
210  """
211  gamma = (other.y() - self.__y) / (other.x() - self.__x)
212  u = -self.__x - other.x() + gamma ** 2
213  v = -self.__y - gamma * (u - self.__x)
214 
215  return self.__class__(u, v)

218  def __double__(self):
219  """
220  Point doubling: add a point to itself.
221 
222  @note This method should not be called directly. Instead use the
223  infix operator @c +, which calls __add__().
224 
225  @note Putting the cases in separate methods makes call profiles more
226  expressive.
227 
228  @see __add__()
229  """
