Detailed Description

An l-torsion group of an elliptic curve for odd l; use the elements() method to have intuitive syntax when working with l-torsion points.

This is a template class that must be instantiated with the elliptic curve. Use it, for example, as follows:

1 # Create the underlying elliptic curve
2 E = elliptic_curves.naive.EllipticCurve( FiniteField(7), 3, 4 )
3 # Instantiate the template; El is a class (here: l-torsion groups of E)
4 El = LTorsionGroup( E )
5 
6 # Construct the group of 3-torsion points and do something with its points
7 for P in El[3].elements():
8  do_something(P)

The l-Torsion points of an elliptic curve are the points of order l. Their coordinates come from the algebraic closure of the elliptic curve's field, which makes them badly suited for computer representation. (Extending the coordinate domain to the algebraic closure makes sure that all points of order l are available; there are always points of order l if l is prime to the field characteristic.)

Note
The l-torsion points are represented implicitly through polynomial arithmetic. Therefore, the list of points in returned by elements() contains only a single entry: the point , where denotes the field of rational functions over the elliptic curve with numerator and denominator polynomials taken modulo the l-th division polynomial .
The class supports only odd torsions greater one because multivariate polynomial arithmetic is unavailable
See Also
Silverman, Joseph H., "The Arithmetic of Elliptic Curves", second edition, Springer, 2009, p. 373
