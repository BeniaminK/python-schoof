Detailed Description

The finite field with elements; the field operations on the element instances support the natural infix syntax and allow mixing in integer arguments.

This is a template class that must be instantiated with the field size. For example:

1 # Instantiate the template; GF23 is a class.
2 GF23 = FiniteField(23)
3 x = GF23(3) # Create a field element: x is the quotient class (3 mod 23)
4 y = GF23(-2) # Equivalently: y = GF23(21) because 21 == -2 (mod 23)
5 z = x**2 + 2*y - x/y # z == 15 (mod 23); note that -2*11 == 1 (mod 23)
6 type(x) is GF23 # This is True

The implementation is held as simple as possible. Its focus lies on ease of understanding, not on performance. Thus it is badly suited for larger computations. The template inherits from rings.quotients.naive.QuotientRing and specializes the _ring template parameter to rings.integers.naive.Integers

Note
The field size must be a prime. The template will, however, accept compound numbers without hesitation. In this case, the class behaves like the quotient ring .
