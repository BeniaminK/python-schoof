Detailed Description

An indexed list of division polynomials over an elliptic curve.

Use it, for example, as follows:

1 # Create the underlying elliptic curve
2 E = elliptic_curves.naive.EllipticCurve( FiniteField(7), 3, 4 )
3 R = CurvePolynomials( E )
4 # Instantiate the list
5 psi = DivisionPolynomialsList( R )
6 
7 # Retrieve some division polynomials
8 p = psi[2] # The second is '2y'
9 q = psi[3] # The third is already longer: 3x^4 + 6Ax^2 + 12Bx -A^2
10 
11 # Use division polynomials for arithmetic on l-torsion points
12 Q = QuotientRing( R, psi[3] )
13 # Do something with Q...

The division polynomials are specific polynomials over an elliptic curve. The l-th divison polynomial has zeros of multiplicity 1 precisely at the finite l-torsion points . Modular polynomial arithmetic then allows computations with the l-torsion points without knowing them explicitly. (Their coordinates come from the algebraic closure of the curve's field; thus they might be badly suited for computer representations.)

An implementation of l-torsion groups using division polynomials is available from elliptic_curves.l_torsion_group.naive.LTorsionGroup.

Note
The implementation lazily constructs the polynomials: the l-th polynomial will be instantiated only if index l is accessed; the polynomial will then be cached.

If the given object supports the interface of elliptic_curves.polynomials.naive.CurvePolynomials, then everything will be fine.

DivisionPolynomials are specific instances of polynomials on the curve. They grow quadratically in degree, which clutters their string representation. Instead of printing the coefficients, print the customary 'psi[l]' where l is the polynomial's index.
