Detailed Description

Call objects represent Function invocations in a CallGraph; they come from pstats.Stats entries and thus provide information on the call counts and execution time.

Lack of detail on the side of the cProfile module, however, makes it impossible to separate specific execution paths. Some information on the timing of a path is available from cumulative_time() because this value is the measured time it took to execute the callee() function body and all functions invoked up to the return.

The class further provides information about a specific invocation path. Use the convenience methods Function.inline_time() and Function.cumulative_time() to retrieve (accumulated) profiling information.
