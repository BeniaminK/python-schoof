Detailed Description

A ring of residue classes of ring elements modulo a fixed ring element; the residue classes (quotient ring elements) support infix notation for ring operations, as well as mixed argument types.

This is a template class that must be instantiated with the source ring and modulus. Use it, for example, as follows:

1 # Instantiate the template; Z4 is a class (here: integer congruence classes)
2 Z4 = QuotientRing( rings.integers.naive.Integers, 4 )
3 x = Z4(1) # Create a residue class: (1 mod 4)
4 y = Z4(3) # Another residue class: (2 mod 4)
5 z = x+y # z is (0 mod 4)
6 z == Z4(2) * 2 # This true because 4 == 0 modulo 4
7 type(z) is Z4 # This is also true

Other examples of QuotientRing template instances are:

1 Z17 = QuotientRing( rings.integers.naive.Integers, 17 ) # Z/17Z
2 
3 # The ring (Z/2Z[x]) / (x^3 + x + 1) is isomorphic to GF8,
4 # the field with 8 elements
5 GF2 = fields.finite.naive.FiniteField( 2 )
6 GF2x = rings.polynomials.naive.Polynomials( GF2 )
7 GF8 = QuotientRing( GF2x, GF2x(1, 1, 0, 1) )

A quotient ring, factor ring, or residue class ring, is a ring , where is the source ring, and denotes the ideal generated by the element . The ring operations carry over to quotient classes in a natural way: if in , then in (by the canonical homomorphism).

With the quotient ring operations being defined in terms of source ring operations, elements of the source ring must implement:

  • __bool__(): Zero testing (True if not zero)
  • __eq__(): Equality testing with the == operator (True if equal)
  • __add__(): Addition with the + operator; self is the left summand
  • __neg__(): Negation with the - unary minus operator (the additive inverse)
  • __mul__(): Multiplication with the * operator; self is the left factor
  • __divmod__(): Division with remainder; self is the dividend (left element)
Note
The implementation emphasizes simplicity over speed; it omits possible optimizations.
The class uses the operand_casting() decorator: other operands in binary operations will first be treated as QuotientRing elements. If that fails, the operation will be repeated after other was fed to the constructor __init__(). If that fails, too, then the operation returns NotImplemented (so that other.__rop__() might be invoked).
See Also
For example, Robinson, Derek J. S., "An Introduction to Abstract Algebra", p. 106.
