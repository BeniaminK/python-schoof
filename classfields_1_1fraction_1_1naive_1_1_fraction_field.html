Detailed Description

A field of formal quotients (fractions) over an integral domain; the field elements support infix notation for field operations, and mixed argument types.

This is a template class that must be instantiated with the underlying integral domain. For example:

1 # Instantiate the template; Q is a class (here: the rational numbers)
2 Q = FractionField( rings.integers.naive.Integers )
3 x = Q(1, 2) # Create a field element: one over two
4 y = Q(2) # Another field element: two, which is two over one
5 z = x*y # z is two over two; no canceling takes place
6 z == Q(4, 4) # This true because 4*2 == 2*4
7 type(z) is Q # This is also true

A formal quotient is an equivalence class of pairs (numerator, denominator) whose entries come from an integral domain (a commutative ring with identity and no zero divisors). Two pairs and are considered equivalent if, and only if, . Observe that this comparison uses multiplication only; since the elements come from a ring and might have no multiplicative inverses, division does not work.

Note
A consequence of omitted canceling is that the elements might grow large.
The class uses the operand_casting() decorator: other operands in binary operations will first be treated as FractionField elements. If that fails, the operation will be repeated after other was fed to the constructor __init__(). If that fails, too, then the operation returns NotImplemented (so that other.__rop__() might be invoked).
See Also
For example, Robinson, Derek J. S., "An Introduction to Abstract Algebra", p. 113.
